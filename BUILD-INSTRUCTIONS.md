# SuperConci — Build Instructions

## Current Phase: Phase 1 — Platform Foundation + Number Blasters Integration

### What Exists
- `src/NumberBlasters.jsx` — the v2 game (standalone, needs refactoring into module)

### Phase 1 Tasks (in order)

#### 1. Project Scaffold
- React 18 + Vite + Tailwind
- PWA config (vite-plugin-pwa: service worker, offline, installable)
- Dexie.js for IndexedDB
- Directory structure per CLAUDE.md architecture
- Vercel deploy config

#### 2. Refactor Number Blasters into Game Module
- Move `src/NumberBlasters.jsx` → `src/games/number-blasters/`
- Split into: `index.js` (module export), `Game.jsx` (main component), `Audio.js` (audio engine)
- Extract the `generateProblem` function to be level-aware
- Add module contract export (see CLAUDE.md "Game Module Contract")
- Wire up to the adaptive engine's skill/level system

#### 3. Adaptive Learning Engine
- `src/engine/AdaptiveEngine.js`
  - Track per-skill accuracy over rolling window (last 20 attempts)
  - Auto-level: promote at >85% mastery, demote at <50%
  - Weak spot targeting: weight problem generation toward weak skills
  - Session pacing: 70% at-level, 20% slightly below (confidence), 10% slightly above (stretch)
- `src/engine/ProgressTracker.js`
  - Persist stats to IndexedDB (Dexie)
  - Calculate overall subject mastery (0-100)
  - Track daily streaks, total problems solved, time played
- `src/engine/AchievementSystem.js`
  - Achievement definitions (first game, 10 streak, 100 problems, etc.)
  - Unlock detection after each session
  - Toast notifications for new achievements

#### 4. World Hub
- `src/hub/WorldHub.jsx`
  - Space-themed hub screen — Conci's ship bridge / command center
  - Game portals as planets/stations (Number Blasters = asteroid belt)
  - Visual progress indicators on each portal
  - Animated background (starfield, ship elements)
- `src/hub/Avatar.jsx`
  - Simple character creator or preset avatars
  - Name input (first launch)
- Profile creation flow on first launch

#### 5. Session Management
- `src/engine/SessionManager.js`
  - Track play time per session
  - Parent-configurable time limits (stored in settings)
  - Gentle "take a break" reminder at limit
  - Session summary screen (what you learned today!)

#### 6. Parent Dashboard (basic)
- PIN-protected access (4-digit code)
- View progress per subject
- View time played
- Sound/music toggles
- Reset progress option (with confirmation)

#### 7. PWA + Deploy
- Service worker caching all game assets
- App manifest (name: SuperConci, icons, theme)
- Offline mode fully functional
- Deploy to Vercel

## Important Notes
- DO NOT generate large JSON files (>30KB) — they stall Claude Code
- All audio is procedural Web Audio API — no audio files
- Everything must work offline
- Touch targets must be LARGE — this is for a 5-year-old
- Celebrate everything — positive reinforcement is the pedagogy
